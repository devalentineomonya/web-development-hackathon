# Web Development Hackathon Project

Welcome to the **Web Development Hackathon** project! This repository is designed to demonstrate collaborative web development practices and build a multi-page website using HTML, CSS, and JavaScript.

## Project Structure

The project directory is structured as follows:

```
web-development-hackathon/
├── css/                  # Stylesheets for each page
│   ├── global.css         # Global styles shared across pages
│   ├── index.css          # Styles for the homepage
│   ├── members.css        # Styles for the members page
│   ├── services.css       # Styles for the services page
│   └── testimonials.css    # Styles for the testimonials page
├── images/                # Project images
├── js/                    # JavaScript files for interactivity
│   ├── index.js           # JS for the homepage
│   ├── members.js         # JS for the members page
│   ├── services.js        # JS for the services page
│   └── testimonials.js    # JS for the testimonials page
├── index.html             # Homepage
├── members.html           # Members page
├── services.html          # Services page
├── testimonials.html       # Testimonials page
└── README.md              # Project documentation
```

---

## Features

- **Modular Design:** Separate CSS and JS files for each page.
- **Scalable Structure:** Designed for easy additions and updates.
- **Collaborative Development:** Encourage teamwork with version control and branching.

## Getting Started

### Prerequisites

Ensure you have the following installed:
- **Git:** Version control for collaboration.
- **Web Browser:** For testing the website locally.

### Clone the Repository

```bash
git clone https://github.com/devalentineomonya/web-development-hackathon.git
cd web-development-hackathon
```

### Run the Project Locally

Open any `.html` file in a web browser to view the corresponding page.

## Contributing Guidelines

We welcome contributions from everyone! Please follow these guidelines to maintain project quality.

### 1. Fork the Repository

Click the "Fork" button on the project page to create a copy of this repository.

### 2. Clone the Forked Repository

```bash
git clone https://github.com/your-username/web-development-hackathon.git
cd web-development-hackathon
```

### 3. Create a Branch

Create a branch with the naming convention `section/[sectionName]`. Replace `[sectionName]` with the part you're working on.

```bash
git checkout -b section/[sectionName]
```

Example:

```bash
git checkout -b section/members
```

### 4. Make Changes and Commit

After making your changes, stage them and commit with a descriptive message.

```bash
git add .
git commit -m "Add feature for members page"
```

### 5. Push Changes

Push the branch to your forked repository.

```bash
git push origin section/[sectionName]
```

### 6. Submit a Pull Request (PR)

Go to the original repository and submit a pull request from your branch.

---

## Collaboration Tips

- Keep commits small and focused.
- Write clear and descriptive commit messages.
- Regularly sync your branch with the main branch.
- Test thoroughly before submitting a PR.
